Model Deployment & Management

Deployments represent named, versioned bindings of models to throughput in a region. Choose a deployment name that encodes environment and intent (e.g., gpt-4o-prod-chat) and track model version changes carefully. Capacity can be scaled up or down to balance latency and cost, and you can run multiple deployments per model for A/B testing.

Lifecycle management includes tagging deployments, rotating them to newer model versions, and decommissioning unused ones. Monitoring throughput and latency helps determine when to increase capacity or shard traffic via API Management. For regulated workloads, maintain separate deployments for PII-safe prompts and general use.

Automate deployment creation using CLI, ARM, or Bicep to ensure consistent naming and quotas across environments. Store metadata in a configuration store so applications can switch deployments without code changes.

ASCII Deployment Flow

Config Repo --> Pipeline --> az deployment create --> Azure OpenAI Deployment
                                 |--> Tag/Version --> Release --> Traffic via API Mgmt

Example Scenario

Example: An e-commerce team runs two deployments, "gpt-4o-prod" and "gpt-4o-abtest", splitting traffic 80/20 through API Management to evaluate prompt templates.

Management Snippet

# Azure CLI: update capacity and list deployments
az cognitiveservices account deployment update \
  -n my-openai -g ai-rg \
  --deployment-name gpt-4o-prod \
  --sku-capacity 200
az cognitiveservices account deployment list -n my-openai -g ai-rg --output table

Real-world Use Case

A media company scales deployment capacity during prime-time traffic spikes and rolls back to previous model versions if quality regressions are detected through human evaluation dashboards.

Beginner Tip & Advanced Note

Beginner Tip: Keep deployment names short and environment-aware, then expose them as configuration values consumed by your app.
Advanced Note: Use deployment slots or parallel deployments for blue-green swaps, and gradually shift traffic using API Management policies with weighted routing.
