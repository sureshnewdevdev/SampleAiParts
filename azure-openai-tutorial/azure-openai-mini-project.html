Mini Project: Secure Azure OpenAI-backed API

This mini project builds a secure backend API that exposes a chat endpoint backed by Azure OpenAI, secured with Azure AD and routed through API Management. It uses a dedicated deployment, managed identity for outbound calls, and integrates a simple RAG step with Azure Search. Deployment is automated via Bicep and GitHub Actions.

The goal is to demonstrate end-to-end governance: infrastructure as code, secretless auth, private networking, and observability. Follow the steps and code snippets to replicate in your environment.

ASCII Project Plan

IaC (Bicep) -> Deploy OpenAI + Search + Key Vault
API Management -> App Service (Managed Identity) -> Azure OpenAI
GitHub Actions -> Deploy API -> Smoke Tests

Example Scenario

Example: Build a secure “policy helper” API that returns grounded answers and requires Azure AD tokens from client applications registered in Entra ID.

Project Snippet

# Bicep: deploy OpenAI and Search
resource openai 'Microsoft.CognitiveServices/accounts@2023-05-01' = {
  name: 'openai-secure'
  location: resourceGroup().location
  kind: 'OpenAI'
  sku: { name: 'S0' }
  properties: { publicNetworkAccess: 'Disabled' }
}
resource search 'Microsoft.Search/searchServices@2022-09-01' = {
  name: 'search-secure'
  location: resourceGroup().location
  sku: { name: 'standard' }
}
